# Scope Monitoring

## Overview
Scope Monitoring helps Product Managers track and analyze changes in project scope, enabling better decision-making and risk management throughout the project lifecycle.

## Key Capabilities
- Create new scope monitoring sessions
- Analyze scope changes using AI
- Track changes, impact assessments, and alerts
- Retry failed sessions

## How to Use
To start, create a new scope monitoring session to capture the initial project scope. Then, analyze any changes using AI to understand their impact. Track changes, impact assessments, and alerts to stay informed. If a session fails, retry it to ensure all data is captured accurately.

## Configuration & Fields
- **Scope Monitoring Session**: Create a new scope monitoring session to define the project scope.
  - **Purpose**: Captures the initial project scope for analysis.
  - **Example**: "New Feature Development for Product X"

- **Data**: Provide the necessary data for analysis.
  - **Purpose**: Enables AI analysis of scope changes.
  - **Example**: "Updated requirements document"

- **Session Reference**: Identify the session for detailed tracking.
  - **Purpose**: Allows tracking of changes, impact assessments, and alerts for a specific session.
  - **Example**: "Session #12345"
