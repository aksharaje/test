# Scenario Modeler

## Overview
Scenario Modeler helps Product Managers create and manage different scenarios for their products, enabling them to analyze and compare various strategies to make informed decisions.

## Key Capabilities
- Create new scenario modeling sessions
- View and manage existing sessions
- Generate scenario variants for comparison
- Access predefined scenario templates

## How to Use
To start using Scenario Modeler, begin by creating a new scenario modeling session based on an existing roadmap. You can then add variants to explore different possibilities and compare outcomes. Utilize the available scenario templates to streamline the process and gain insights into potential strategies.

## Configuration & Fields
- **Scenario Session Create**
  - **Scenario Data**: Data for creating a new scenario session
  - **Example**: Market research results for a new product launch

- **User ID**
  - **Purpose**: Identifies the owner or user associated with the session
  - **Example**: JohnDoe123

- **Roadmap Session ID**
  - **Purpose**: References a specific roadmap for scenario sessions
  - **Example**: 98765

- **Scenario Variant Create**
  - **Session ID**: Identifies the session for creating a new scenario variant
  - **Data**: Data for creating a new scenario variant
  - **Example**: Variant details for a pricing strategy

- **Template Name**
  - **Purpose**: Specifies the predefined template to base a new variant on
  - **Example**: "Market Expansion Plan"

- **Scenario Variant Update**
  - **Variant ID**: Identifies the variant to be updated
  - **Data**: Updated data for the scenario variant
  - **Example**: Revised sales projections

- **Variant ID**
  - **Purpose**: Identifies the specific variant for deletion or modification
  - **Example**: 12345

- **Available Scenario Templates**
  - **Purpose**: Provides access to predefined scenario templates
  - **Example**: "Competitor Analysis"
