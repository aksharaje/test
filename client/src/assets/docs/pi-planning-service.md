# Pi Planning

## Overview
Pi Planning helps Product Managers effectively plan and manage their product development cycles. It enables PMs to streamline their planning process and ensure alignment across teams.

## Key Capabilities
- Create new planning sessions
- View and analyze existing sessions
- Track progress and milestones
- Manage holiday configurations

## How to Use
To start using Pi Planning, begin by creating a new planning session where you can outline your product development goals and timelines. You can then add relevant data and collaborate with your team to ensure everyone is on the same page. Use the tool to track progress, make adjustments as needed, and keep stakeholders informed throughout the development cycle.

## Configuration & Fields
- **Session**: The planning session where you define your product development goals and timelines.
  - **Purpose**: To set a clear direction for the upcoming development cycle.
  - **Example**: Create a session for the next quarter to plan feature releases and prioritize tasks.

- **Integration ID**: An identifier for integrating external data or systems into your planning session.
  - **Purpose**: To link relevant data from other tools or sources to enrich your planning process.
  - **Example**: Use an integration ID to pull in customer feedback data from a survey tool.

- **Data**: Additional information or details to include in the planning session.
  - **Purpose**: To provide context or specific requirements for the development tasks.
  - **Example**: Include user stories and acceptance criteria for a new feature in the data section.

- **Holiday Configurations**: Settings to manage holidays and non-working days within the planning timeline.
  - **Purpose**: To account for non-working days and ensure accurate scheduling of tasks.
  - **Example**: Configure holidays to exclude weekends and public holidays from the development timeline.
