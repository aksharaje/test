# Flow Manager

## Overview
The Flow Manager helps Product Managers streamline the management of project workflows, ensuring efficient execution and tracking of tasks. It empowers PMs to oversee project progress and make informed decisions.

## Key Capabilities
- Create new project flows
- Update existing project details
- Start project execution
- Track project executions
- Delete unnecessary project flows

## How to Use
To begin, start by creating a new project flow with relevant details. Update project information as needed, then initiate project execution. Track the progress of project executions to ensure tasks are completed efficiently. Finally, remove any redundant project flows.

## Configuration & Fields
- **Session**: Represents the current work session.
  - **What it's for**: Identifies the context in which actions are performed.
  - **Example**: Morning planning session for a specific project.
  
- **ID**: Unique identifier for a project flow.
  - **What it's for**: Distinguishes between different project flows.
  - **Example**: Project ID 12345 for the marketing campaign.

- **Data**: Information related to the project flow.
  - **What it's for**: Contains details about the project.
  - **Example**: Includes project name, start date, and assigned team members.

- **Context**: Additional context for project execution.
  - **What it's for**: Provides specific details for executing a project.
  - **Example**: Contextual information for executing a marketing campaign.

- **Execution ID**: Identifier for a project execution.
  - **What it's for**: Tracks the progress of a specific project execution.
  - **Example**: Execution ID 6789 for the current marketing campaign run.
