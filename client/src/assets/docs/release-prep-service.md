# Release Preparation

## Overview
Release Preparation helps Product Managers organize and plan their product releases efficiently. It streamlines the process of creating release artifacts, managing release notes, and tracking decisions and technical debt items.

## Key Capabilities
- Create new release prep sessions
- Get all sessions and filter them by user
- Manage release artifacts from Story Generator, Epic Creator, and Feature Creator
- Update release notes, decisions, and technical debt items
- Export release notes, decision logs, and technical debt inventory

## How to Use
To start using Release Preparation, begin by creating a new release prep session. Then, you can add and manage release artifacts, track decisions, and handle technical debt items. Finally, export the necessary documentation in markdown format for easy sharing and collaboration.

## Configuration & Fields
- **Session Reference**: Identifies the specific session for managing release artifacts and information.
  - **Purpose**: Helps organize and track all activities related to a particular release.
  - **Example**: Q3 2022 Release Planning Session

- **Owner**: Indicates the user responsible for the session.
  - **Purpose**: Assigns ownership and accountability for the release preparation activities.
  - **Example**: John Doe

- **Include Released**: Determines whether to include artifacts already in a release for debugging purposes.
  - **Purpose**: Allows users to control the visibility of released artifacts during the preparation phase.
  - **Example**: True

- **Note Data**: Contains the updated information for a release note.
  - **Purpose**: Enables users to modify and maintain accurate release notes.
  - **Example**: Revised feature descriptions

- **Decision Data**: Holds the updated details for a decision made during the release preparation.
  - **Purpose**: Facilitates tracking and updating decisions throughout the release process.
  - **Example**: Revised timeline for feature implementation

- **Debt Item Data**: Stores the new technical debt item details for manual creation.
  - **Purpose**: Supports the identification and management of technical debt within the release.
  - **Example**: Refactor legacy code in Module X
