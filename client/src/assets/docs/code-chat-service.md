# Code Chat

## Overview
Code Chat enables Product Managers to effectively communicate with developers, share knowledge, and manage coding sessions collaboratively. It streamlines the process of discussing code-related topics and resolving queries efficiently.

## Key Capabilities
- Start a coding session
- Retrieve code knowledge bases
- Send messages within a coding session
- Manage and track coding sessions
- Update session knowledge bases

## How to Use
To begin, initiate a coding session to discuss a specific code-related topic or query with your team. Retrieve relevant code knowledge bases to reference during the session. Utilize the messaging feature to communicate within the session seamlessly. Manage and track all coding sessions to ensure progress and productivity. Update session knowledge bases as needed to keep information current.

## Configuration & Fields
- **Session**: Represents a work session for discussing code-related topics.
  - **What it's for**: Identifies the specific coding session being created or accessed.
  - **Example**: "Code Review Meeting"

- **Data**: Contains information related to the coding session.
  - **What it's for**: Stores details such as agenda, objectives, and action items for the coding session.
  - **Example**: "Agenda: Review new feature implementation"

- **Session ID**: Unique identifier for each coding session.
  - **What it's for**: Helps differentiate between multiple coding sessions and facilitates easy retrieval.
  - **Example**: "Session ID: 12345"

- **User Message**: Text input for sending messages within a coding session.
  - **What it's for**: Allows users to communicate, share insights, or ask questions during the coding session.
  - **Example**: "Please review the latest code changes."

- **Connected Knowledge**: Refers to the knowledge bases associated with the coding session.
  - **What it's for**: Links relevant resources or documentation that support the discussion during the session.
  - **Example**: "Connected Knowledge: API documentation, design guidelines"
