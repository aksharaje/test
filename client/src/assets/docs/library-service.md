# Library Management

## Overview
The Library Management tool helps Product Managers efficiently manage their library of resources, enabling them to easily access, organize, and update information related to their projects and research.

## Key Capabilities
- Create new books
- List all available books
- Retrieve specific book details
- Generate content for a book
- Delete books

## How to Use
To get started, begin by creating a new book in the library. You can then list all existing books to review the available resources. When needed, you can retrieve detailed information about a specific book, generate content for it, or delete books that are no longer relevant.

## Configuration & Fields
- **Session**: Represents your current work session.
  - **What it's for**: Identifies the context in which you are interacting with the library.
  - **Example**: Your current work session while managing a specific project.
  
- **User ID**: Optional field to specify the user associated with a book.
  - **What it's for**: Links a book to a particular user if needed.
  - **Example**: Assigning a book to a team member for review.

- **Book ID**: Unique identifier for each book in the library.
  - **What it's for**: Helps distinguish one book from another.
  - **Example**: Identifying a specific research document in the library.

- **Knowledge Base ID**: Indicates the knowledge base related to a book.
  - **What it's for**: Connects the book to a specific knowledge repository.
  - **Example**: Linking a book to a database of industry insights.

- **Commit Hash**: String representing a specific version or update.
  - **What it's for**: Tracks changes made to the book's content.
  - **Example**: Documenting revisions made to a research report.

- **Change Summary**: Brief description of the modifications made.
  - **What it's for**: Provides a summary of the alterations for reference.
  - **Example**: Noting the key updates made to a book's content.
