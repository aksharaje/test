# Experience Gap Analyzer

## Overview
The Experience Gap Analyzer helps Product Managers identify and bridge gaps in user experience, leading to improved product performance and customer satisfaction.

## Key Capabilities
- Create new gap analysis sessions
- View session details and related data
- Manage and update identified gaps
- Run background gap analysis pipelines
- Reorder gaps within the roadmap tiers

## How to Use
To start, create a new gap analysis session to define the scope of your analysis. Add gaps manually or let the system generate them for you. Review and update gap details as needed. Run the gap analysis pipeline to generate insights and reorder gaps to prioritize actions effectively.

## Configuration & Fields
- **Analysis Type**
  - **Purpose**: Specifies the type of analysis being conducted.
  - **Example**: "Competitor Comparison"

- **Your Journey ID**
  - **Purpose**: Identifies the user journey being analyzed.
  - **Example**: "Onboarding Process"

- **Comparison Journey ID**
  - **Purpose**: Optional field to compare against a different user journey.
  - **Example**: "Previous Product Version"

- **Analysis Name**
  - **Purpose**: Provides a descriptive name for the analysis session.
  - **Example**: "Q3 User Experience Review"

- **User ID**
  - **Purpose**: Identifies the user associated with the analysis session.
  - **Example**: "Product Manager John Doe"

- **Connected Knowledge Base**
  - **Purpose**: Links the analysis to relevant knowledge bases for reference.
  - **Example**: "Customer Feedback Repository"

- **Analysis Parameters**
  - **Purpose**: Additional parameters to customize the analysis process.
  - **Example**: "Include User Surveys"

- **Title**
  - **Purpose**: Describes the identified gap in a concise manner.
  - **Example**: "Lack of Mobile Responsiveness"

- **Description**
  - **Purpose**: Provides detailed information about the identified gap.
  - **Example**: "Key features not optimized for mobile devices"

- **Category**
  - **Purpose**: Categorizes the gap for better organization and analysis.
  - **Example**: "Usability"

- **Impact Score**
  - **Purpose**: Quantifies the impact of the identified gap on user experience.
  - **Example**: "7.5"

- **Urgency Score**
  - **Purpose**: Rates the urgency of addressing the identified gap.
  - **Example**: "8.2"

- **Effort Score**
  - **Purpose**: Estimates the effort required to resolve the identified gap.
  - **Example**: "5.6"

- **Stage Name**
  - **Purpose**: Indicates the stage of product development related to the identified gap.
  - **Example**: "Design Phase"
