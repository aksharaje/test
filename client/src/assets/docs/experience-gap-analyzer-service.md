# Experience Gap Analysis Tool

## Overview
The Experience Gap Analysis Tool helps Product Managers identify and address gaps in user experience, enabling them to enhance product usability and customer satisfaction.

## Key Capabilities
- Create new gap analysis sessions
- Retrieve session details
- List and filter sessions
- Add and delete gaps
- Run gap analysis pipeline
- Reorder gaps in the roadmap

## How to Use
To start, create a new gap analysis session to define the scope of analysis. Then, add relevant gaps and provide details such as title, description, and impact scores. Once gaps are added, run the gap analysis pipeline to generate insights. You can also reorder gaps in the roadmap to prioritize actions effectively.

## Configuration & Fields
- **Analysis Type**: Type of analysis being conducted
  - **What it's for**: Specifies the focus of the analysis (e.g., usability, performance)
  - **Example**: Conducting a usability analysis to improve user interactions
  
- **Your Journey ID**: Identifier for the main user journey
  - **What it's for**: Links the analysis to a specific user journey within the product
  - **Example**: Analyzing gaps in the onboarding process (Your Journey ID: 123)
  
- **Comparison Journey ID**: Identifier for a secondary user journey for comparison
  - **What it's for**: Allows comparison between different user journeys
  - **Example**: Contrasting the checkout process with the registration process
  
- **Analysis Name**: Name to identify the analysis session
  - **What it's for**: Provides a descriptive title for the analysis session
  - **Example**: Naming the analysis session "Mobile App Redesign Analysis"
  
- **Knowledge Base IDs**: IDs of related knowledge bases for reference
  - **What it's for**: Links relevant knowledge bases to the analysis for additional insights
  - **Example**: Referring to knowledge bases on user personas and market trends
