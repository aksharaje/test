# Experience Gap Analyzer

## Overview
The Experience Gap Analyzer helps Product Managers identify and address gaps in user experience to improve product quality and customer satisfaction. By analyzing sessions, PMs can gain insights into areas for enhancement and prioritize improvements effectively.

## Key Capabilities
- Create new gap analysis sessions
- View session details and related data
- Add and delete gaps for analysis
- Run gap analysis pipeline for insights
- Reorder gaps within tiers for better organization

## How to Use
Start by creating a new gap analysis session to kick off your analysis. Add gaps manually or run the gap analysis pipeline to generate insights. Review session details and related data to make informed decisions. Use the drag-and-drop feature to reorder gaps for better visualization and prioritization.

## Configuration & Fields
- **Analysis Type**: Type of analysis being conducted
  - **Purpose**: Specifies the focus of the analysis
  - **Example**: User Flow, Feature Usage

- **Your Journey**: Reference ID for the primary user journey
  - **Purpose**: Identifies the main user flow for analysis
  - **Example**: Onboarding Process (ID: 123)

- **Comparison Journey**: Reference ID for a secondary user journey (optional)
  - **Purpose**: Allows comparison between different user flows
  - **Example**: Beta Version (ID: 456)

- **Analysis Name**: Name for the analysis session (optional)
  - **Purpose**: Provides a descriptive title for the session
  - **Example**: Q4 Product Update Analysis

- **Owner**: User responsible for the analysis session
  - **Purpose**: Designates the owner of the session
  - **Example**: John Doe

- **Connected Knowledge Base**: Relevant knowledge base for analysis
  - **Purpose**: Links to additional resources or information
  - **Example**: UX Best Practices

- **Title**: Title of the gap being added
  - **Purpose**: Describes the gap in the user experience
  - **Example**: Checkout Process Confusion

- **Description**: Detailed description of the identified gap
  - **Purpose**: Provides context and specifics about the gap
  - **Example**: Users find the checkout button hard to locate

- **Category**: Category of the gap (e.g., Usability, Performance)
  - **Purpose**: Classifies the type of gap for analysis
  - **Example**: Navigation

- **Impact Score**: Numeric value indicating the impact of the gap
  - **Purpose**: Quantifies the severity of the gap
  - **Example**: 7.5

- **Urgency Score**: Numeric value indicating the urgency to address the gap
  - **Purpose**: Prioritizes the gap based on urgency
  - **Example**: 8.2

- **Effort Score**: Numeric value indicating the effort required to fix the gap
  - **Purpose**: Estimates the resources needed to address the gap
  - **Example**: 5.9

- **Stage Name**: Name of the stage where the gap exists (optional)
  - **Purpose**: Identifies the specific stage impacted by the gap
  - **Example**: Checkout Page
